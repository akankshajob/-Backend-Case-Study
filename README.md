# Backend Case Study – Inventory Management (B2B SaaS)

This repository contains a backend engineering case study for a B2B inventory management platform StockFlow.

All work is documented and implemented in **`Casestudy.py`** using Python and Flask, with explanations written as code comments.

## What’s Covered

- Part 1: Code Review & Debugging  
  Reviewed and fixed a product creation API by adding validation, transaction safety, SKU uniqueness checks, and error handling.

- Part 2: Database Design
  Designed a relational schema supporting multiple companies, warehouses, inventory tracking, suppliers, and product bundles, with SQL DDL included.

- Part 3: API Implementation
  Implemented a low-stock alerts API that considers product-specific thresholds, recent sales activity, multiple warehouses, and supplier information.

## Focus Areas

Backend best practices data consistency, scalability, and business-driven decision making.
